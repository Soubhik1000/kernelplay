Critical Architecture Details
Scene Component Registries:
javascriptthis._rigidbody2D = [];      // 2D physics
this._rigidbody3D = [];      // 3D physics
this._colliders = [];        // 2D colliders
this._colliders3D = [];      // 3D colliders
this._renderers = [];        // Render components
this._cameras = [];          // Camera components
this.primaryCamera = null;   // Active camera
Spatial Grid:

_grid2D / _grid3D - HashMap of grid cells
_gridCellSize = 128 - Cell size (2-4x object size)
Rebuilt every physics frame
Collision: O(n) instead of O(n²)

3D Collision (Y-up coordinate system):
javascript// CRITICAL: Y-up means higher Y = higher position
if (centerA > centerB) {  // A is ABOVE B
  posA.y += overlapY + EPS;  // Push UP (positive)
  if (rbA.velocity.y < 0) rbA.velocity.y = 0;  // Stop falling
  rbA.isGrounded = true;
} else {  // A is BELOW B (ceiling)
  posA.y -= overlapY + EPS;  // Push DOWN (negative)
  if (rbA.velocity.y > 0) rbA.velocity.y = 0;  // Stop jumping
}

javascriptget bounds() {

    this.rebuildBounds();
  }
  return this._bounds;  // Cached object (no allocation)
}

5. Recent Bugs & Fixes
Just Solved ✅

3D collision pass-through - Fixed Y-up coordinate comparisons (centerA > centerB for landing)
Objects sinking through ground - Fixed push direction (posA.y += overlapY for Y-up)
Ceiling collision wrong - Fixed push direction (posA.y -= overlapY when hitting ceiling)
Camera not switching - Need to update scene.primaryCamera manually, not just isPrimary
BoxCollider3D undefined transform - Added init() method to cache transform reference
Dirty flag disabled - User commented out dirty check, causing performance issues

Key Learnings

3D uses Y-up (Three.js) vs 2D Y-down (Canvas) - comparisons reversed
Gravity in Y-up: velocity.y -= gravity * dt (subtract, not add)
Always call transform._dirty = true after modifying position in collision resolution
Component registries require exact string match in _registerComponent() switch case
Cached references break if components destroyed - get fresh or handle null checks


Variable Names to Remember

scene.primaryCamera - Current active camera
scene._colliders3D - 3D collider registry
this._props - ScriptComponent injected props
EntityReference.entityId - ID for serialization
camera.viewBounds - Camera visible area (for culling)
rb.isGrounded - Reset every _physicsStep(), set during collision

Architecture Choices

ECS not pure - Components can have logic (Unity-style, not strict ECS)
Spatial grid rebuilt every frame - Not persistent (simpler, fast enough)
Object pooling automatic - instantiate() handles it, no manual pool management
Prop injection over constructors - Scripts receive props object, auto-injected as this.propName
ID-based references - For scene editor, resolved at runtime via scene.findById()

Status: Engine is production-ready for 2D/3D games. Focus now on tooling (editor, serialization) and content systems (audio, particles).
